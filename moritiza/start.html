Installation

Server Requirements

Make sure your server meets the following requirements:

  • PHP >= 7.2.0
  • Ctype PHP Extension
  • JSON PHP Extension
  • Mbstring PHP Extension
  • Curl PHP Extension
  • PDO PHP Extension

MoriTiza Composer

MoriTiza utilizes Composer to manage its dependencies. So, make sure you have Composer installed on your machine.

Via Composer Create-Project

composer create-project --prefer-dist mortezanasiri/moritiza:dev-master blog

Or Download latest MoriTiza.zip from GitHub!

Download MoriTiza.zip

Configuration

Directory Permissions

After installing MoriTiza, you may need to configure some permissions. Directories within the core/storage should be writable by your web server or MoriTiza will not run.

.htaccess configuration

Change RewriteBase in .htaccess to specific dir that MoriTiza installed on.

RewriteBase /moritiza

Example

If your diroctory like this: /var/www/blog

You may change RewriteBase from RewriteBase /moritiza to RewriteBase /blog

.env configuration

Set DB_DATABASE, DB_USERNAME, DB_PASSWORD, So that MoriTiza can access your database.

DB_DATABASE=
DB_USERNAME=
DB_PASSWORD=

Notice!

After development is over you should change APP_DEBUG=true to APP_DEBUG=false

Initializing

When all configurations done, you must run init command.

Init command generate APP Key and create initial tables in database!

php moritiza init

Notice!

Init command just run at first time. Don't run it again!

Hello, MoriTiza

Finally, Go to your project url and now you must see welcome page!
